## What was done this session

### Schema gap confirmed and fixed

The V4 schema had `enrolments` linked to `schools` only. ISHS runs Mon/Tue/Thu with 36/45/39 students respectively — completely separate cohorts. Without a link from enrolment to session_slot, `get_enrolments_for_session` would return all 120 ISHS students for any ISHS session, tripling the order count.

The previous session (007) flagged this as a V5 concern and proposed a system-prompt workaround. That workaround does not solve the over-counting — the fix is structural.

**Fix applied: `enrolment_session_slots` junction table (V4-OPT-07)**

```sql
CREATE TABLE enrolment_session_slots (
    enrolment_id    bigint NOT NULL REFERENCES enrolments(id) ON DELETE CASCADE,
    session_slot_id bigint NOT NULL REFERENCES session_slots(id) ON DELETE RESTRICT,
    joined_date     date NOT NULL DEFAULT CURRENT_DATE,
    left_date       date,
    PRIMARY KEY (enrolment_id, session_slot_id),
    CHECK (left_date IS NULL OR left_date >= joined_date)
);
```

With indexes:
- `idx_enrolment_session_slots_active_session` ON `(session_slot_id) WHERE left_date IS NULL`
- `idx_enrolment_session_slots_enrolment` ON `(enrolment_id)`

**Deployed to Supabase.** Table exists, 0 rows (seed script not yet written).

### Key query pattern for `get_enrolments_for_session`

```sql
SELECT e.id, e.student_name, e.student_year_level, e.opted_out_of_catering,
       array_agg(dt.name) FILTER (WHERE dt.name IS NOT NULL) AS dietary_tags
FROM enrolment_session_slots ess
JOIN enrolments e ON e.id = ess.enrolment_id
JOIN schools sc ON sc.id = e.school_id
LEFT JOIN enrolment_dietary_tags edt ON edt.enrolment_id = e.id
LEFT JOIN dietary_tags dt ON dt.id = edt.dietary_tag_id
WHERE ess.session_slot_id = %(session_slot_id)s
  AND ess.left_date IS NULL
  AND e.opted_out_of_catering = false
  AND e.current_period_end_date IS NULL  -- active enrolment
  AND NOT EXISTS (
      SELECT 1 FROM absences a
      WHERE a.enrolment_id = e.id AND a.absence_date = %(session_date)s
  )
GROUP BY e.id, e.student_name, e.student_year_level, e.opted_out_of_catering
```

Dietary students absent on the day still get a meal (CLAUDE.md rule) — handle that at order composition level, not in this query.
