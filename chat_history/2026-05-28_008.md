## What was done this session

### Seed misalignment discovered and fixed

`seed_feedback_history.py` had `WEEK_STARTS` anchored on `2026-04-21`, which is a **Tuesday**, not Monday. Every `_session_date(week_start, day_of_week)` call was therefore 1 weekday late. All 55 seeded orders had `session_date` on the wrong weekday.

**Impact of bug (now fixed):**
- `get_sessions_needing_orders` idempotency guard never fired — weekday filter eliminated the misaligned orders first, so the agent would have re-triggered ordering for any historically-ordered session.
- `get_feedback_for_session` (queries by `session_slot_id, session_date`) would have returned empty for all historical sessions.
- `compute_rolling_mean` was unaffected — uses `feedback.caterer_id + submitted_at` directly (V4-OPT-04).

**Fix:**
```python
# Before (all Tuesdays — wrong):
WEEK_STARTS = [datetime.date(2026, 4, 21), datetime.date(2026, 4, 28), ...]

# After (all Mondays — correct):
WEEK_STARTS = [
    datetime.date(2026, 4, 20),   # Week 1
    datetime.date(2026, 4, 27),   # Week 2
    datetime.date(2026, 5,  4),   # Week 3
    datetime.date(2026, 5, 11),   # Week 4
    datetime.date(2026, 5, 18),   # Week 5
]
```

Cascade scope verified before truncate:
- `order_lines.order_id` → CASCADE (was 0 rows; seed doesn't recreate — no loss)
- `feedback.order_id` → CASCADE (55 rows; seed recreates 55)
- `inbound_email_records.related_order_id` → SET NULL (0 rows)
- `outbound_emails.related_order_id` → SET NULL (0 rows)

`seed_feedback_history.py` confirmed as the **sole writer** to `orders` (grep across all of `src/ingest/`). Truncated + re-seeded. Alignment verified via hard assert using `EXTRACT(ISODOW FROM session_date)` vs `session_slots.day_of_week`: **55 orders checked, 0 misaligned**.

| `get_year_level_exclusions(school_id, session_date)` | ❌ | Filters `enrolment_id IS NULL AND year_levels_excluded != '{}'`; returns `list[int]` not text-parsed |

**`upsert_absence` semantics — DO NOTHING (record why):**
`ON CONFLICT (enrolment_id, absence_date) DO NOTHING`. The tools_reference explicitly says "ignores duplicate" — first notification is the record of truth. V4 has no absence-amendment workflow. DO UPDATE was initially implemented but rejected: it contradicts the spec authority, and introduced a provenance asymmetry (`received_at` preserved from first email, `source_email_message_id` overwritten from second — "when we first heard" and "which email told us" pointing at different notifications). DO NOTHING keeps all fields consistent. Because `DO NOTHING` + `RETURNING id` yields no row on conflict, the implementation does an explicit `SELECT id` fallback to return the existing id.

**Exclusion scoping model (V4):**

| school_id | enrolment_id | year_levels_excluded | Meaning |
|---|---|---|---|
| NOT NULL | NULL | `[]` | Full-school — all year levels |
| NOT NULL | NULL | `[10, 12]` | Year-level — only those year levels |
| NOT NULL | NOT NULL | any | Per-enrolment — single student |

## Locked conventions (on record for all future modules)

| Convention | Value |
|---|---|
| DB import | `from src.ingest.db import get_conn` |
| `get_conn()` contract | psycopg3 context manager: commits on clean exit, rolls back on exception, closes in both cases. Write fns call `conn.commit()` explicitly (context-manager exit is then a no-op). Read-only fns: no commit. |
| Money | Integer cents, never floats |
| Timestamps | `timestamptz` in DB; timezone-aware `datetime` in Python |
| Timezone | `ZoneInfo("Australia/Brisbane")` — UTC+10, no DST |
| Naive Postgres time | Localise to Brisbane when building `datetime` objects |
| Order lead time | `settings.order_hours_before_session` (=72) — never hardcode |
| T-72h anchor | `session_slots.start_time`, not `dinner_time` |
| Urgency enum | `'urgent'`, `'notable'`, `'informational'`, `'none'` — match Postgres `step_urgency` enum exactly |
| jsonb columns | `psycopg.types.json.Jsonb(value)` wrapper |
| Architecture | Tools are deterministic and dumb; agent reasons. Gaps surface as data for escalation — never silently filtered in SQL. |
