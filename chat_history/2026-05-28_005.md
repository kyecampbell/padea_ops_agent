## DB implications flagged to user (action required)

Two schema additions are needed for the changes just documented. Tables are empty so these are free ALTER TABLE statements.

### 1. Free-text allergy field — `enrolments` table

The doc now describes an "other" free-text allergy entry that escalates to the operator. Currently `enrolments` has no column for this. Proposed addition:

```sql
ALTER TABLE enrolments ADD COLUMN other_allergy_notes text;
```

This stores the parent's free-text allergy entry. The escalation logic reads this field and flags it when non-null at term-start email processing.

### 2. Manager checklist — `feedback` table

The doc now defines 5 checklist boolean questions. The current schema has:
- `food_on_time boolean` ✅
- `visibly_wrong boolean` ✅ (maps to "packaging intact / nothing visibly wrong")

Missing (need to add):
```sql
ALTER TABLE feedback ADD COLUMN correct_count_received boolean;
ALTER TABLE feedback ADD COLUMN correct_dietary_delivered boolean;
ALTER TABLE feedback ADD COLUMN food_temperature_ok boolean;
```
