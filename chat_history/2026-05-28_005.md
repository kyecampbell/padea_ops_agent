## Known schema gap (for order composition tool)

V4 enrolments link to `school_id` only. ISHS has students in 3 separate sessions (Mon, Tue, Thu) but the DB cannot distinguish which cohort attends which slot. The order composition tool must account for this: it will see all ISHS enrolments as "active" regardless of which session is being ordered for.

Mitigation: the `session_tutor_assignments` table gives tutors per slot; the agent's system prompt should instruct it to order for all active enrolments at a school for any given slot unless business rules say otherwise.

**V5 trigger:** add `session_slot_id` to `enrolments` table.
