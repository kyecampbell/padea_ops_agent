### STEP 2 — compose_session_order VO fallback (APPROVED)

File changed: `src/tools/orders.py`

Two new private functions added before `compose_session_order`:

**`_auto_pick_vo_meal(enrolment_id, caterer_id)`**

SQL: find active VO item for caterer where item_tags ⊇ (student_tags − {'vegetarian'}).
```sql
SELECT mi.id, mi.name, mi.price_cents
FROM menu_items mi
WHERE mi.caterer_id = %s
  AND mi.active = true
  AND mi.has_vegetarian_option = true
  AND NOT EXISTS (
      SELECT 1
      FROM enrolment_dietary_tags edt
      JOIN dietary_tags dt ON dt.id = edt.dietary_tag_id
      WHERE edt.enrolment_id = %s
        AND dt.name != 'vegetarian'
        AND edt.dietary_tag_id NOT IN (
            SELECT midt.dietary_tag_id
            FROM menu_item_dietary_tags midt
            WHERE midt.menu_item_id = mi.id
        )
  )
ORDER BY mi.id LIMIT 1
```
Returns `{menu_item_id, name, price_cents}` or None.

**`_vo_safe_for_enrolment(menu_item_id, enrolment_id)`**

Same predicate as `_auto_pick_vo_meal`, executed as defense-in-depth after
selection. NOT hardcoded True. Any uncovered tag other than `vegetarian` returns
False. Mirrors `item_is_safe_for_enrolment` but excludes the vegetarian tag from
the check.

**Selection chain in `compose_session_order`** (per-student loop):
```
request → rotation → auto_pick_dietary_meal → _auto_pick_vo_meal → escalate
```
New code:
```python
variant = None   # initialised per student

# ... request / rotation paths unchanged (variant stays None) ...

if meal is None:
    meal = auto_pick_dietary_meal(enrolment_id, caterer_id)
    if meal:
        source = "dietary_auto_pick"
        safe = item_is_safe_for_enrolment(meal["menu_item_id"], enrolment_id)
    else:
        vo_meal = _auto_pick_vo_meal(enrolment_id, caterer_id)
        if vo_meal:
            meal = vo_meal
            source = "dietary_auto_pick"
            variant = "vegetarian_option"
            safe = _vo_safe_for_enrolment(vo_meal["menu_item_id"], enrolment_id)
        else:
            escalations.append("URGENT: no safe meal ...")
            continue
```
