### BUILD 1 + 2 ‚Äî `src/tools/weekly_summary.py`
Two new functions:

`generate_weekly_summary(caterer_id, week_of, as_of=None) -> dict`
- ISO week normalisation: `week_start = week_of - timedelta(days=week_of.weekday())`
- already_completed guard: checks `outbound_emails` for existing `weekly_consolidated_summary` row for same caterer √ó ISO week; returns `{"already_completed": True}` with no mutations
- GST normalisation: `grand_total_cents = round(total_cost_cents √ó 1.1)` for `price_includes_gst=False`; `gst_amount_cents = grand_total - total_cost`
- MOQ: variety_count ‚Üí moq_applicable (only if count ‚àà {4,5,6}); shortfall written to LAST order (by session_date ASC) as absolute SET (idempotent)
- Quality: `compute_rolling_mean(caterer_id, 4/12, as_of)`; `sustained_decline = mean_4w < quality_floor AND (mean_12w - mean_4w) >= quality_decline_threshold`
- Returns: caterer_id/name/email, week_start/end, total_items, total_cost_cents, grand_total_cents, gst_amount_cents, delivery_fee_cents (per delivery, informational), sessions_count, moq_applicable, moq_floor_applied, moq_variance_cents, mean_4w, mean_12w, sustained_decline, session_breakdown, already_completed

`compose_weekly_summary_email(caterer_id, summary) -> str`
- Read-only; no DB writes
- Sections: week header, per-session breakdown, cost/GST/TOTAL DUE block, delivery fee (informational ‚Äî not in total), MOQ floor note if applied, quality ratings, sustained-decline alert
- Demo mode prepend: `[DEMO ‚Äî Intended for: {caterer_email}]`
