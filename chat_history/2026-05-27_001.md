## Key context to carry forward

- **Schema**: `docs/v3_final/Schema/v3_schema.sql` is the authoritative DDL. Run it verbatim in Supabase.
- **Python version**: Use `.venv` (Python 3.12 via uv). `source .venv/bin/activate` before running code.
- **Email mode**: `email_mode: demo` in `runtime_config.yaml`. The `demo_email_address` is `kyec898@gmail.com`. Never flip to production during development.
- **V3-MP-01**: Zero operational margin — `standard_contingency_meals: 0`. No tutor buffer, no contingency lines, except the 10% exclusion buffer.
- **Enrolment date queries**: Always pass an explicit date. Never query "currently active" — always "active on [date]".
- **Approved vs full menu**: Rotation uses approved set only. Meal requests (tutor-filed) use the full dietary-safe menu.
