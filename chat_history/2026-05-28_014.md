### loop.py design declaration — round 2 (full detail) — PENDING APPROVAL

Expanded declaration covers all 6 points at full depth. Key decisions in the design:

#### Tool registry
- **21 tools exposed to Claude**, **6 functions excluded** (loop-internal/infrastructure):
  - Excluded: `create_agent_run`, `complete_agent_run`, `log_agent_step`, `check_and_resolve_crashed_runs` (loop startup/teardown, called in Python only), `create_order` (internal to `compose_session_order` — exposing risks duplicate order creation), `consume_meal_request` (internal to `compose_session_order`)
  - All 21 Python signatures confirmed ✓ from real module source, not inferred
  - Full JSON `input_schema` specified for each tool

#### Type coercion in dispatch
Date/datetime parameters arrive as JSON strings and must be coerced to Python types before the real function call. A `_PARAM_TYPES` dict maps `(tool_name, param_name) → python_type` for all 12 affected parameters across the 21 tools. Coercion: `"YYYY-MM-DD"` → `date.fromisoformat()`, ISO datetime string → `datetime.fromisoformat()` then `.replace(tzinfo=BRISBANE)` if naive.

#### Three observation shapes — (b) vs (c) distinction is critical
- **(a) Success with data**: plain serialised result dict or list, `is_error` absent
- **(b) "Nothing found" / shortfall / business signal**: `null` or dict with shortfall field, `is_error` absent — tool ran correctly, data says something. Agent uses business logic.
- **(c) Exception**: `{"error": true, "type": "...", "message": "..."}`, `is_error: True` — tool failed to run. Agent uses system-fault escalation.

(b) vs (c) is preserved at the Anthropic API boundary via the `is_error` flag. A `null` from `get_absence` (no absence filed) is `is_error: False`. A `ValueError` from `get_session_slot(999)` (not found) is `is_error: True`. Conflating these is a real bug; the design explicitly separates them.

#### `compose_session_order` as composite tool
Confirmed it **already exists** in `src/tools/orders.py`. Functions it orchestrates internally: `get_enrolments_for_session` (which applies cohort filtering), `get_meal_request`, `consume_meal_request`, `get_next_rotation_meal`, `item_is_safe_for_enrolment`, `auto_pick_dietary_meal`, `create_order`. Decision log visibility comes from: (a) `tool_output_full` stores the full result including `order_lines[].source` showing how each student got their meal; (b) Claude calls `add_note(urgency="urgent")` for each item in the returned `escalations` list, producing separate urgent log rows. Per-student individual tool calls would exhaust the 15-call cap for any realistic cohort (20+ students).

#### `add_note` meta-tool (loop-backed, no Python function in `src/tools/`)
The only way for Claude to assign urgency to a reasoning step. When dispatched, `log_agent_step` is called INSIDE the dispatch function (not in the outer loop) so the outer loop skips its standard log call for `add_note`. `step_index` is owned by the outer loop and incremented once per block — no collision possible. DB's `UNIQUE (run_id, step_index)` is the hard guard. The `label` field (not "add_note") becomes `tool_name` in `agent_steps` — operators see e.g. "sustained_decline_detected" not "add_note".

#### Haiku — NOT an architecture change, NOT deferred
"Haiku for routing" means `classify_inbound_email` (a Gmail tool, not yet built) will use Haiku internally as its sub-LLM call. The loop is always Sonnet 4.6. There is nothing to defer — Haiku enters the codebase inside `gmail.py` when that tool is built.

#### Hard cap
Checked via `call_count >= max_tool_calls_per_run` BEFORE each dispatch. On cap hit: log `max_calls_reached` step at `urgency="urgent"`, feed `is_error: True` tool_result for every remaining block in the response (Anthropic API requires a result for every `tool_use_id`), set `cap_hit = True` and `continue` (not `break`) through the block loop. After the block loop: make one final model call (not counted toward cap) for Claude to summarise what was completed vs what remains. Call `complete_agent_run(run_id, f"[CAP HIT at {N}/{cap}] {final_text}")`.
