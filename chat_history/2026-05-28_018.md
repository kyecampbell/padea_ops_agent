Cap behaviour:
- Cap checked **BEFORE** dispatch on each tool-use block
- When cap is hit: one `urgent` `max_calls_reached` step logged; remaining blocks get `is_error` tool_result blocks (loop uses `continue`, not `break`); assistant + tool_results appended; an uncounted final no-tools summary call is made; `complete_agent_run` called with `"[CAP HIT at N/N]"` message

- `is_error` dispatch results log with `urgency="urgent"`
- `json.dumps` wrapped → `SerialiseError` produces an `is_error` block; run continues (does not crash)
- `add_note` is logged inside `dispatch()`; loop skips it from the step-logging path
